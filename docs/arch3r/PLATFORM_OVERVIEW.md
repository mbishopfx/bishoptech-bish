# ARCH3R Platform Overview

ARCH3R is a shared AI workspace for organizations that need one system for thinking, collaborating, approving, and executing. It is intentionally broader than a developer tool and more operational than a standalone chat product.

## What ARCH3R does

- Shared AI chat with organization history, model switching, shared threads, and archived threads.
- Knowledge-backed work through uploads, Google-connected content, and organization-scoped retrieval.
- Huddles for live collaboration with room presence, notes, reactions, and project-linked sessions.
- Approvals and operator controls for actions that should not execute without a human gate.
- Project management and internal ticket triage inside the same workspace.
- Campaign lanes for social publishing, voice operations, and SMS operations.
- A plugin marketplace so orgs activate only the tools they actually want in the left toolbar.
- A local listener bridge so cloud coordination can hand real implementation work to Gemini or Codex on a trusted machine.

## Why it matters

Most teams are currently splitting work across:

- one or more AI chat subscriptions
- Slack or email for coordination
- a project tracker
- a call tool
- a social scheduler
- a spreadsheet or CRM export process
- a local coding assistant or terminal agent

ARCH3R collapses those into one system with one workspace model and one organization memory layer.

## What makes it different

### 1. It is shared by default

Threads, projects, tickets, campaign logs, huddles, and operator views are organization features, not one-user silos.

### 2. It is tool-activated instead of app-sprawl driven

The marketplace and activation model let one deployment serve many customer shapes:

- collaboration-only
- collaboration + social
- collaboration + voice
- collaboration + voice + SMS + projects

### 3. It can actually build

ARCH3R is not limited to cloud chat. The Local Listener can hand work to a local Gemini or Codex runtime and return progress back to the thread.

### 4. It is policy-aware

Model access, approvals, connectors, plugins, and org entitlements live under one organization control surface.

## Core workspace lanes

- `Chat`: shared AI collaboration, handoffs, shared visibility, model switching
- `Huddle`: live rooms for voice collaboration and note capture
- `Marketplace`: plugin activation and readiness
- `Projects`: private-by-default kanban collaboration boards
- `Ticket Triage`: internal issue/request intake that can promote into projects
- `Social`: outbound publishing and scheduling
- `Voice`: Vapi-backed calling campaigns and transcript workflows
- `SMS`: Twilio-backed messaging campaigns
- `Operator`: organization dashboard for tool, member, and campaign activity

## Ideal customer profiles

- Agencies managing client operations and implementation in one workspace
- Sales teams running outreach, follow-up, and reporting in one system
- Internal ops teams that need approvals, campaigns, notes, and handoffs
- Founders and lean teams that want one AI workspace instead of multiple tool subscriptions
- Enterprise operators who need a cloud control plane plus local execution paths
